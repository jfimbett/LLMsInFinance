## Discounted Cash Flow (DCF)

:::: {.columns}

::: {.column width="60%"}
- **Core Principle**: Value = Present value of future cash flows
- **Key Components**:
  - Forecast period (typically 5-10 years)
  - Terminal value
  - Discount rate (WACC)
- **Advantages**:
  - Theoretically sound
  - Forward-looking
  - Captures long-term value creation
- **Challenges**:
  - Sensitivity to assumptions
  - Forecast accuracy
  - Terminal value dominance
:::

::: {.column width="40%"}
$$\text{Enterprise Value} = \sum_{t=1}^{n} \frac{FCF_t}{(1+WACC)^t} + \frac{TV}{(1+WACC)^n}$$

$$TV = \frac{FCF_{n+1}}{WACC - g}$$

Where:
- $FCF_t$ = Free Cash Flow in year t
- $WACC$ = Weighted Average Cost of Capital
- $TV$ = Terminal Value
- $g$ = Perpetual growth rate
:::

::::

## Multiples-Based Valuation

:::: {.columns}

::: {.column width="60%"}
- **Core Principle**: Value based on similar assets/companies
- **Common Multiples**:
  - P/E (Price-to-Earnings)
  - EV/EBITDA (Enterprise Value to EBITDA)
  - EV/Revenue
  - P/B (Price-to-Book)
  - Industry-specific metrics
- **Advantages**:
  - Market-based
  - Simple to apply
  - Reflects current sentiment
- **Challenges**:
  - Peer selection
  - Market inefficiencies
  - Differences in accounting practices
:::

::: {.column width="40%"}
$$\text{Implied Value} = \text{Multiple}_{\text{peer}} \times \text{Metric}_{\text{target}}$$

Example:
$$\text{EV}_{\text{target}} = \text{Median EV/EBITDA}_{\text{peers}} \times \text{EBITDA}_{\text{target}}$$

- Requires careful selection of:
  - Comparable companies
  - Appropriate multiples
  - Adjustment factors
:::

## The Look-Ahead Bias Challenge

:::: {.columns}

::: {.column width="60%"}
- **Definition**: Using future information not available at forecast time
- **Critical issue with LLMs**:
  - Training data includes past events presented as historical facts
  - Model may "know" outcomes that weren't available at decision point
- **Manifestations**:
  - "Predicting" known past events
  - Incorporating market reactions that followed events
  - Referencing subsequent strategic shifts
:::

::: {.column width="40%"}
- **Mitigation strategies**:
  - Explicit temporal context setting
  - Point-in-time data retrieval
  - Vintage database access
  - Prompt engineering with time constraints
  - Validation against contemporary forecasts
  - Critical evaluation of predictions
:::

def cross_impact_matrix(variables):
    """
    Generate cross-impact matrix for scenario
    
    Parameters:
    variables - Key scenario variables
    
    Returns:
    Matrix of impact relationships and strengths
    """
    matrix = {}
    
    for v1 in variables:
        matrix[v1] = {}
        for v2 in variables:
            if v1 != v2:
                # LLM assessment of relationship
                impact = assess_impact(v1, v2)
                matrix[v1][v2] = impact

def identify_comparables(target_company, 
                        candidate_pool,
                        dimensions,
                        weights):
    """
    LLM-powered comparable company selection
    
    Parameters:
    target_company - Company to find peers for
    candidate_pool - Universe of potential peers
    dimensions - Similarity dimensions to assess
    weights - Importance weights for dimensions
    
    Returns:
    Ranked comparables with similarity scores
    """
    comparables = []
    
    for candidate in candidate_pool:
        # Calculate similarity across dimensions
        similarity = {}
        for dim in dimensions:
            score = assess_similarity(
                target_company, candidate, dim)
            similarity[dim] = score
        
        # Calculate weighted similarity score
        weighted_score = sum(
            similarity[d] * weights[d] 
            for d in dimensions)
        
        comparables.append({
            "company": candidate,
            "score": weighted_score,
            "dimension_scores": similarity
        })
    
    # Sort by overall similarity
    comparables.sort(key=lambda x: x["score"], 
                    reverse=True)
    
    return comparables

def extract_entity_network(text):
    """
    Extract entities and their relationships
    
    Parameters:
    text - Financial document text
    
    Returns:
    Entity network with relationship types
    """
    # Extract entities
    entities = extract_entities(text)
    
    # Identify relationships
    relationships = []
    for e1 in entities:
        for e2 in entities:
            if e1 != e2:
                relation = identify_relationship(text, e1, e2)
                if relation:
                    relationships.append({
                        "source": e1,
                        "target": e2,
                        "type": relation["type"],
                        "context": relation["context"]
                    })
    
    return {"entities": entities, 
            "relationships": relationships}

    for doc in documents:
        # Extract entities and relationships
        extraction = extract_entity_network(doc.text)
        
        # Add to graph with document as source
        for entity in extraction["entities"]:
            add_or_update_node(graph, entity, doc)
            
        for rel in extraction["relationships"]:
            add_or_update_edge(graph, rel, doc)
    
    return graph
